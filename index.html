Git-GitHub

  • create a new repository on the command line - создайте новый репозиторий в командной строке

    • git init
    • git add README.md
    • git commit -m "first commit"
    • git remote add origin https://github.com/vintsan/Git-GitHub.git
    • git push -u origin main

  • or push an existing repository from the command line - или запустите существующий репозиторий из командной строки

    • git remote add origin https://github.com/vintsan/Git-GitHub.git
    • git branch -M main
    • git push -u origin main

Коммиты

  • git status
  • git add . - добавляет все файлы в stage
  • git commit -m "comment" - commit = запись
  • git log / git log --oneline - инфа по коммитам
  • git remote -v - информация о репозитории (ссылка на наш репозиторий)
  • git push [rep_link] [branch_name] - например [origin - ссылка на наш репозиторий] [master - название ветки]

Ветки в Git. Создание веток и их отправка на GitHub

  • git branch - просмотр созданных веток
  • git branch develop - создание ветки "develop"
  • git checkout develop - переход на ветку "develop"
  • git remote -v - информация о репозитории (ссылка на наш репозиторий)
  • git push [rep_link] [branch_name] - сохранение на github (например [origin - ссылка на наш репозиторий] [master - название ветки])

Слияние веток при помощи действия pull request

На сайте github:

  • жмем кнопку "pull request",жмем кнопку "New pull request", выбираем ветки (из develop в master), жмем кнопку "create pull request", далее можно указать название и описание, мем кнопку "create pull request", в "Reviewers" на сайте справа можно выбрать людей которые могут просматривать код,
  • на сайте в закладке "Files changed" можно посмотреть все изменения в ветке developer (к каждому изменению можно давать коментарии или доп. задачи) - сохранить коментарии нажать "Finish your review" с общим коментарием и нажать "Submit review"
  • после каждого исправления жмем кнопку "Resolve conversation"
  • после того как все исправления и баги будут пофикшены можно делать слияние веток!!!
  • для переноса изменений из одной ветки в другую жмем кнопку "Merge pull request" и подтверждаем "Confirm merge"

В VS Code

  • git branch - просмотр созданных веток
  • Переходим на основную ветку "git checkout master"
  • git merge develop - рабочую ветку "develop" соединили с основной "master"
